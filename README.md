
🚀 Linux 优化脚本 

一个专为 Debian/Ubuntu/CentOS 服务器设计的交互式系统优化脚本。集成了内核升级、BBR 加速、安全加固、Swap 配置、应用优化等 15 项核心功能，旨在一键提升服务器性能与安全性。

⚠️ 重要提示：本脚本为纯交互模式。请务必先运行脚本，待菜单出现后再输入数字选择功能。切勿在命令行直接输入数字（如 15），否则会导致 command not found 错误。

✨ 核心功能
编号   功能模块   说明
1   系统自动更新   更新软件源及所有已安装包至最新稳定版

2   升级最新稳定内核   自动检测并安装官方最新稳定内核 (需重启)

3   SSH 安全定制   修改默认端口、禁用 Root 密码登录、密钥优化

4   基础参数优化   调整文件描述符、进程数、网络堆栈参数

5   BBR 拥塞控制   开启 TCP BBR v2/v3，大幅提升网络吞吐

6   智能 Swap 配置   根据内存大小自动计算并创建 Swap 分区，防 OOM

7   防火墙配置   自动配置 UFW/Firewalld，仅开放必要端口

8   启动项服务优化   禁用不必要的系统服务，加快启动速度

9   应用环境优化   针对 Nginx/MySQL/PHP 等常见环境的参数调优

10   安全加固   禁用 IPv6 (可选)、隐藏内核版本、防 SYN 攻击

11   系统垃圾清理   清理缓存、旧内核、日志文件，释放磁盘空间

12   监控工具安装   一键安装 Htop, Netdata 或 Btop 等监控工具

13   配置回滚   救命功能：一键还原优化前的系统配置备份

14   DNS 优化   切换为高性能公共 DNS (Cloudflare/Google/阿里)

15   🚀 全量优化   推荐：按顺序自动执行上述所有优化项

📦 快速开始

下载脚本
请使用 wget 或 curl 将脚本下载到本地（推荐做法，避免管道执行导致的函数丢失问题）：

方法 A: 使用 wget (推荐)
wget -O optimize.sh https://raw.githubusercontent.com/vonyifan/linux-auto-optimize-script/main/Linux.sh

方法 B: 使用 curl
curl -o optimize.sh https://raw.githubusercontent.com/vonyifan/linux-auto-optimize-script/main/Linux.sh

赋予执行权限
chmod +x optimize.sh

运行脚本
./optimize.sh

❓ 常见问题 (FAQ)

Q1: 为什么我输入数字提示 command not found？
A: 这是因为你没有先运行脚本。
❌ 错误做法：在终端直接输入 15。
✅ 正确做法：先输入 ./optimize.sh 启动脚本，等到菜单出现后，再在提示符下输入 15。

Q2: 可以直接用 curl ... | bash 运行吗？
A: 强烈不建议。
由于本脚本包含大量自定义函数，管道执行方式容易导致 Bash 解析时序错误，引发 function not found 或逻辑混乱。请务必先下载保存为文件，再执行。

Q3: 优化后需要重启吗？
A: 
如果你选择了 选项 2 (升级内核) 或 选项 5 (BBR)，通常建议重启服务器 (reboot) 以生效。
其他大部分优化即时生效，无需重启。
脚本在执行完毕后会明确提示是否需要重启。

Q4: 如何查看日志或回滚？
A: 
日志位置：/root/optimize_script_YYYYMMDD_HHMMSS.log
备份位置：/root/optimize_backup_YYYYMMDD_HHMMSS/
回滚：再次运行脚本，选择 选项 13 即可自动恢复。
