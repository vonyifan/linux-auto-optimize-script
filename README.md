# Linux 生产环境全维度优化脚本 (增强版)



---

## 📖 项目简介

本脚本是一个专为 **CentOS/RHEL** 和 **Debian/Ubuntu** 系列 Linux 服务器设计的自动化运维工具。它集成了系统更新、内核升级、SSH 安全加固、网络参数调优（BBR）、Swap 智能管理、防火墙配置、应用优化（Nginx/MySQL）等 14 个核心模块。

脚本采用**交互式菜单**设计，支持单项执行或一键全量优化。内置完善的**备份与回滚机制**，确保在生产环境操作的安全性。

### ✨ 核心优势
*   🛡️ **安全第一**：修改 SSH 端口前强制风险确认，敏感信息（密钥）不记录日志，关键操作前自动备份。
*   🚀 **性能提升**：自适应调整内核参数，自动启用最新 BBRv3/v2 拥塞控制算法，显著提升网络吞吐。
*   🔄 **安全可靠**：所有配置文件修改均支持一键回滚，防止配置错误导致系统不可用。
*   🧠 **智能适配**：根据 CPU 核数、内存大小、磁盘空间自动计算最优参数，兼容 UEFI/Legacy 启动模式。

## 🚀 快速开始

### 1. 系统要求
*   **操作系统**: CentOS 7+/8+/9+, RHEL, Ubuntu 18.04+, Debian 9+
*   **权限**: 必须使用 `root` 用户运行
*   **网络**: 需连接互联网以下载更新包和源

### 2. 一键运行
直接使用 `curl` 下载并执行（推荐先下载查看）：

```bash
# 方式 A: 直接运行（慎用，建议先查看代码）
curl -sSL https://raw.githubusercontent.com/vonyifan/linux-auto-optimize-script/main/Linux.sh | bash

# 方式 B: 下载后运行（推荐）
wget -O optimize.sh https://raw.githubusercontent.com/vonyifan/linux-auto-optimize-script/main/Linux.sh
chmod +x optimize.sh
./optimize.sh
```

### 3. 操作指南
运行脚本后将进入交互菜单：
1.  输入数字 `1-14` 执行单项优化任务。
2.  输入 `15` 执行**全量优化**（推荐新机器初始化时使用）。
3.  输入 `13` 可在配置出错时**一键回滚**到初始状态。
4.  输入 `0` 退出脚本。

## ⚠️ 重要注意事项 (必读)

在使用本脚本前，请务必阅读以下风险提示：

1.  **SSH 端口修改风险**：
    *   若选择修改 SSH 端口（模块 3），脚本会检测当前是否为远程连接。
    *   **警告**：修改端口会导致当前会话断开！请务必在**另一个终端窗口**测试新端口连通性 (`ssh -p <新端口> root@IP`) 成功后，再关闭当前窗口。
    *   脚本会自动尝试配置防火墙放行新端口，但仍需确保云服务商的安全组已放行。

2.  **内核升级风险**：
    *   升级内核（模块 2）后**必须重启服务器**才能生效。
    *   虽然脚本会自动设置默认启动项，但建议在重启前通过 VNC/控制台确认 Grub 菜单配置正确，以防无法远程连接。

3.  **备份机制**：
    *   脚本每次运行都会在 `/root/optimize_full_YYYYMMDD_HHMMSS/` 目录下备份关键配置文件。
    *   若优化后出现异常，请使用 **模块 13 (一键配置回滚)** 或直接从备份目录手动恢复。

4.  **磁盘空间检查**：
    *   创建 Swap 文件（模块 6）前，脚本会自动检查磁盘剩余空间。若空间不足，操作将自动终止，防止填满磁盘导致系统崩溃。

5.  **生产环境建议**：
    *   建议在 `screen` 或 `tmux` 会话中运行脚本，防止网络波动导致脚本中断。
    *   首次使用建议先在测试机运行全量流程，熟悉各项配置后再应用于生产环境。

## 🛠️ 技术细节

### 自适应参数策略
脚本会根据硬件资源动态计算最优值：
*   **文件描述符 (`nofile`)**: `CPU 核数 * 8192` (最小 65535, 最大 1048576)
*   **Swap 大小**:
    *   内存 ≥ 32G: 建议禁用或保留 8G
    *   16G ≤ 内存 < 32G: 16G
    *   内存 < 16G: 内存的 2 倍 (最大 16G)
*   **BBR 版本**: 自动检测内核版本，优先启用 BBRv3 (Kernel 6.2+) > BBRv2 (5.18+) > 经典 BBR。

### 支持的发行版
*   **RedHat 系**: CentOS 7, 8, 9, RHEL, AlmaLinux, Rocky Linux
*   **Debian 系**: Ubuntu 18.04 - 24.04, Debian 9 - 12

## ❓ 常见问题 (FAQ)

**Q: 脚本运行失败怎么办？**
A: 查看日志文件 `/root/optimize_script_YYYYMMDD_HHMMSS.log` 获取详细错误信息。如果是网络问题，请检查服务器能否访问外网。

**Q: 修改 SSH 端口后连不上了！**
A: 
1. 不要关闭当前会话（如果还没断）。
2. 检查云厂商控制台的“安全组”是否放行了新端口。
3. 如果已断开且无法连接，需通过云厂商提供的 **VNC/控制台** 登录服务器，使用模块 13 回滚配置或手动修改 `/etc/ssh/sshd_config`。

**Q: 如何卸载脚本做的修改？**
A: 运行脚本，选择选项 `13) 一键配置回滚`，脚本会自动恢复所有备份的配置文件并删除新增的配置项。

**Q: 支持 ARM 架构 (如 AWS Graviton, 树莓派) 吗？**
A: 目前主要针对 x86_64 架构优化。ARM 架构下内核包名和 Grub 配置可能不同，请谨慎使用“内核升级”模块，其他模块通常兼容。
